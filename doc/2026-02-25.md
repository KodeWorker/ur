## Architecture planning

Designed 7 tech tracks for a local-first, cross-platform Python agent sandbox:

| Track | Tech |
|---|---|
| 1 ‚Äî Sandbox Runtime | subprocess (default) ‚Üí Docker (opt-in) ‚Üí WASM (future) |
| 2 ‚Äî Agent Engine | `anthropic` + `litellm`, hand-rolled async loop, `pydantic` schemas |
| 3 ‚Äî Tool Suite | `playwright`, `httpx`, `aiofiles`, sandbox |
| 4 ‚Äî Memory & State | `aiosqlite` (SQLite), optional `chromadb` |
| 5 ‚Äî Orchestration | `asyncio` ‚Üí `arq` ‚Üí `AgentBus` protocol seam |
| 6 ‚Äî Observability | `structlog`, token tracking, optional OTel + Langfuse |
| 7 ‚Äî Interface | `typer` + `rich` CLI, optional `fastapi` REST/SSE |

Key cross-platform decisions:
- `psutil` replaces Unix-only `resource` module for process limits
- `platformdirs` replaces hardcoded `~/.ur` for OS-correct data dirs
- `pathlib.Path` throughout, no string paths
- `asyncio.create_subprocess_exec` instead of `shell=True`

## Phase 1 implementation ‚Äî "Can it think?"

### Files created

```
pyproject.toml
.env.example
src/ur/__init__.py               # __version__ = "0.1.0"
src/ur/config.py                 # Settings via pydantic-settings + platformdirs
src/ur/agent/models.py           # Message type alias, UsageStats dataclass
src/ur/agent/session.py          # AgentSession ‚Äî messages, usage, lifecycle
src/ur/agent/loop.py             # Core agentic loop (async generator)
src/ur/llm/client.py             # LLMClient + CompletionStream wrapping litellm
src/ur/memory/db.py              # SQLite schema init, get_db context manager
src/ur/memory/session_store.py   # save_session, list_sessions, get_session_messages
src/ur/main.py                   # Typer CLI: ur run / ur chat / ur history / ur init
```

No empty `__init__.py` in sub-packages ‚Äî namespace packages used throughout.

### Design decisions

- **Dumb loop, smart tools**: the agentic loop is ~25 lines; no tool dispatch in Phase 1
- **Session lifecycle managed by CLI**: `loop.run()` does not call `session.complete()` ‚Äî callers own the lifecycle
- **Messages are plain dicts**: litellm/OpenAI format `{"role": ..., "content": ...}` ‚Äî no wrapper class
- **Bug found and fixed**: `INSERT OR IGNORE` on `messages` table would duplicate rows on repeated saves (no unique constraint). Fixed to `DELETE + re-INSERT` on each save

### CLI commands

```
ur init                    # create data dirs + db
ur run "<task>"            # single task, streaming output
ur run "<task>" -m <model> # override model
ur chat                    # multi-turn REPL
ur history                 # list sessions
ur history <id>            # show messages for a session
```

## Ollama support

Added hosted Ollama as a second provider:

- `Settings.ollama_base_url` (env: `UR_OLLAMA_BASE_URL`, default: `http://localhost:11434`)
- `LLMClient.stream()` passes `api_base` when model starts with `ollama` ‚Äî covers both `ollama/` and `ollama_chat/`

```ini

UR_MODEL=ollama_chat/qwen2.5
```

## Provider-selective test markers

Tests specific to one provider are skipped when running against the other:

```python
# conftest.py
skip_if_not_anthropic = pytest.mark.skipif(not TEST_MODEL.startswith("anthropic"), ...)
skip_if_not_ollama    = pytest.mark.skipif(not TEST_MODEL.startswith("ollama"), ...)
```

Decorated tests:

| Test | Marker |
|---|---|
| `test_anthropic_api_key_from_env` | `@skip_if_not_anthropic` |
| `test_llm_client_passes_api_key_when_set` | `@skip_if_not_anthropic` |
| `test_llm_client_omits_api_key_when_empty` | `@skip_if_not_anthropic` |
| `test_non_ollama_model_does_not_pass_api_base` | `@skip_if_not_anthropic` |
| `test_ollama_base_url_default` | `@skip_if_not_ollama` |
| `test_ollama_base_url_override` | `@skip_if_not_ollama` |
| `test_ollama_model_passes_api_base` | `@skip_if_not_ollama` |
| `test_ollama_chat_model_passes_api_base` | `@skip_if_not_ollama` |
