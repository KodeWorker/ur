## Phase 3 design updates

### plan/phase3.md

- **TUI backend**: adopted ftxui v5.0.0 (FetchContent) instead of a hand-rolled
  line-based TUI. Rationale: component model scales across Phase 3–6 without a
  rewrite (collapsible reasoning, tool-call display, context status bar, streaming).
- **Reasoning block**: changed from "dimmed" to `ftxui::Collapsible` — collapsed
  by default, user can expand inline. Updated chat loop step 3d and Reasoning
  Display section accordingly.
- **Chat loop 3c**: spinner runs on a background thread, posts `Event::Custom`
  to the ftxui `ScreenInteractive` at ~100 ms intervals.
- **Chat loop 3d**: `"reason"` role messages are persisted to DB and shown as a
  collapsible block; only `"assistant"` messages go back into the context window.
- **Chat loop 3g**: plain-text exit commands dropped; `/exit` is the only exit
  path — all `/`-prefixed input is intercepted as TUI slash commands.
- **Persona updater — meaningful-turn filter** added:
  - Skip extraction unless user message > 50 chars AND context ≥ 6
    user/assistant messages.
  - Thresholds are conservative starting points; open questions documented for
    later refinement.
- **`ur history <id>`**: `[reason]` row shown between `[user]` and `[assistant]`
  when a thinking block was captured.
- **Acceptance criteria**: removed stale "if key loaded" qualifiers (plaintext
  mode was dropped in Phase 2.1).

  for explicit user approval before execution.
  - `ur chat`: ftxui modal block (tool name + formatted args); `y`/`n` inline.
  - `ur run`: prints to stderr, reads `y/n` from stdin; non-interactive stdin
    treated as rejection.
  - Approval/rejection recorded as `"tool_audit"` message role.

### Design decisions

- `Tui` is an abstract interface — `FtxuiTui` for production, `MockTui` (defined
  locally in test files) for tests. No ftxui dependency in test binaries.
- `Chat::strip_think` and `Chat::build_window` are `public static` so they can
  be unit-tested directly without running the full event loop.
- `PersonaUpdater::maybe_update` swallows all exceptions — persona extraction is
  best-effort and must never crash the chat loop.
- `generate_id()` duplicated as a file-local helper in `chat.cpp` (same pattern
  as `runner.cpp`). Consolidation to a shared utility deferred to a later phase.

## `HttpProviderTest` regression fix — ctest 100 % passing

Three tests that were green in Phase 2.1 failed after Phase 3 scaffolding was
added:

| Test | Symptom |
|------|---------|
| `CompletePostsToCorrectEndpoint` | "connection failed" / Bus error |
| `CompleteReturnsAssistantContent` | "connection failed" |
| `CompleteIncludesAuthHeader` | "connection failed" / SEGFAULT |

### Root cause

`HttpProvider::complete()` called `client.set_read_timeout(0, 0)` when
`UR_LLM_READ_TIMEOUT` was not set, intending "wait indefinitely". In httplib,
`select()` with `{sec=0, usec=0}` is a non-blocking poll — it returns
immediately with "no data" before the loopback test server thread can process
the request and write a response. The bug was latent from Phase 2; Phase 3's
additional sources (ftxui, chat, persona_updater) changed thread scheduling
enough to make it consistently reproducible.

The ASAN `stack-use-after-return` report was the decisive clue: the server
thread was executing the test handler (writing to a stack variable in
`TestBody`) *after* `TestBody` had returned — proving the server *did* receive
the request but the client had already given up due to the 0-second read poll.

### Fixes

- **`src/ur/llm/http_provider.cpp`** — skip `set_read_timeout` entirely when
  `UR_LLM_READ_TIMEOUT` is absent; httplib's 300 s default is used instead of
  a 0-second poll.
- **`tests/unit/test_http_provider.cpp`** — replaced the manual
  `is_running()` polling loop with `svr_.wait_until_ready()` (httplib's
  built-in spin-wait); also removed the now-unused `<chrono>` include.

`ctest --test-dir build`: **100 % passed, 0 failed** (35 pass, 17 GTEST_SKIP
stubs); all pre-commit hooks green.

## Streaming inference — `ur run` and `ur chat`

Both commands now stream tokens to the display as they arrive instead of
waiting for the full completion.

### Provider layer

- **`src/ur/llm/provider.hpp`** — added `TokenCallback` type alias
  (`std::function<void(const std::string&)>`) and `stream()` pure virtual
  method alongside `complete()`.
- **`src/ur/llm/http_provider.hpp/.cpp`** — implemented `stream()`:
  sends `"stream": true`, uses httplib `Request.response_handler` +
  `content_receiver` to parse SSE lines (`data: {...}\n\n`), and calls
  `token_cb` / `reasoning_cb` for `delta.content` and
  `delta.reasoning_content` chunks respectively.

### Runner

- **`src/ur/agent/runner.hpp/.cpp`** — `run()` now accepts optional
  `token_cb` / `reasoning_cb` (default `nullptr`). When either is set,
  wraps them with accumulators and calls `provider.stream()`; otherwise
  falls back to `provider.complete()`. Full accumulated text is persisted
  to the DB either way.

### `ur run`

- **`src/ur/cli/command.cpp`** — `cmd_run` passes a `token_cb` that writes
  each chunk to `stdout` with `flush`; a final `'\n'` is printed after the
  stream ends.

### `ur chat` (TUI)

- **`src/ur/cli/tui.hpp`** — added `print_response_chunk(chunk)` to the
  `Tui` interface. `print_response()` remains for non-streaming paths.
- **`src/ur/cli/tui.cpp`** (`FtxuiTui`) — `print_response_chunk()`: first
  chunk creates a new history entry backed by a `shared_ptr<string>` buffer;
  subsequent chunks append to the buffer and `PostEvent` a redraw. The
  component's `Renderer` lambda captures the shared pointer so it always
  renders the latest text without rebuilding the component tree.
  `print_user()` resets `streaming_content` to `nullptr` so each turn
  gets a fresh component.
- **`src/ur/agent/chat.cpp`** — inference loop switched from
  `provider.complete()` to `provider.stream()`: spinner stops on the first
  content token, content chunks go to `tui.print_response_chunk()`, and
  reasoning chunks are buffered and shown as a collapsible block after the
  stream ends. `<think>` fallback stripping is applied only when no
  `reasoning_content` field was received.

### Also fixed

- `UR_LLM_READ_TIMEOUT=0` in `.env` now correctly means "use httplib default
  (300 s)"; previously it triggered an immediate `select({0,0})` poll that
  returned before the server could respond.

### Mock updates

All `MockProvider` classes in tests (`test_runner`, `test_chat`,
`test_persona_updater`) implement the new `stream()` override (delivers the
canned response as a single chunk). `MockTui` in `test_chat` adds
`print_response_chunk`, `print_user`, `print_error`, `set_status`, and
`system_prompt()` stubs.

pre-commit hooks green.
