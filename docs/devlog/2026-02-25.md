## Architecture planning

Designed 7 tech tracks for a local-first, cross-platform Python agent sandbox:

| Track | Tech |
|---|---|
| 1 — Sandbox Runtime | subprocess (default) → Docker (opt-in) → WASM (future) |
| 2 — Agent Engine | `litellm`, hand-rolled async loop, `pydantic` schemas |
| 3 — Tool Suite | `playwright`, `httpx`, `aiofiles`, sandbox |
| 4 — Memory & State | `aiosqlite` (SQLite), optional `chromadb` |
| 5 — Orchestration | `asyncio` → `arq` → `AgentBus` protocol seam |
| 6 — Observability | `structlog`, token tracking, optional OTel + Langfuse |
| 7 — Interface | `typer` + `rich` CLI, optional `fastapi` REST/SSE |

Key cross-platform decisions:
- `psutil` replaces Unix-only `resource` module for process limits
- `platformdirs` replaces hardcoded `~/.ur` for OS-correct data dirs
- `pathlib.Path` throughout, no string paths
- `asyncio.create_subprocess_exec` instead of `shell=True`

### Files created

```

src/ur/agent/session.py          # AgentSession — messages, usage, lifecycle
src/ur/agent/loop.py             # Core agentic loop (async generator)
src/ur/llm/client.py             # LLMClient + CompletionStream wrapping litellm
src/ur/memory/db.py              # SQLite schema init, get_db context manager
src/ur/memory/session_store.py   # save_session, list_sessions, get_session_messages
src/ur/main.py                   # Typer CLI: ur run / ur chat / ur history / ur init
```

No empty `__init__.py` in sub-packages — namespace packages used throughout.

### Design decisions

- **Dumb loop, smart tools**: the agentic loop is ~25 lines; no tool dispatch in Phase 1
- **Session lifecycle managed by CLI**: `loop.run()` does not call `session.complete()` — callers own the lifecycle
- **Messages are plain dicts**: litellm/OpenAI format `{"role": ..., "content": ...}` — no wrapper class
- **Bug found and fixed**: `INSERT OR IGNORE` on `messages` table would duplicate rows on repeated saves (no unique constraint). Fixed to `DELETE + re-INSERT` on each save

### CLI commands

```
ur init                    # create data dirs + db
ur run "<task>"            # single task, streaming output
ur run "<task>" -m <model> # override model
ur chat                    # multi-turn REPL
ur history                 # list sessions
ur history <id>            # show messages for a session
```

## LLMClient refactor — provider at construction time

`LLMClient` previously detected the provider on every `stream()` call via
`self.settings.model.startswith(...)`. Refactored to detect once at `__init__`:

```python
class Provider(str, Enum):
    GEMINI = "gemini"
    OLLAMA = "ollama"
    OTHER  = "other"

class LLMClient:
    def __init__(self, settings: Settings) -> None:
        self.settings = settings
        self.provider = self._detect_provider(settings.model)   # detected once

    @staticmethod
    def _detect_provider(model: str) -> Provider: ...
```

Added 5 new tests in `test_llm_client.py` under `# Provider detection`:
`test_detect_provider_gemini`, `test_detect_provider_ollama`,
`test_detect_provider_ollama_chat`, `test_detect_provider_other`,
`test_llm_client_stores_provider_at_init`.
