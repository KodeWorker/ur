# 2026-03-15

## Phase 4 — Tool System Architecture and Abstractions

### Architecture design

Full Phase 4 architecture designed before any implementation. Key decisions:

**`Message` struct extension** — `ToolCallEntry`, `tool_calls` vector, and
`tool_call_id` field added to `provider.hpp`. Two constructors added
(`Message(role, content)` and default) to preserve all existing `{"role",
"content"}` call sites across the codebase.

**`ServerInfo::supports_vision`** — `bool supports_vision = false` added to
`ServerInfo`. Used by `Loader` to gate `read_image` registration — tool is
disabled even if `tools.json` enables it when the model does not report vision
support.

**Tool-calling loop uses `complete()` not `stream()`** — tool call JSON must be
fully assembled before execution; streaming cannot deliver it incrementally.
Final text response turns stream normally. Inner loop capped at 10 iterations
to guard against infinite tool-call cycles.

**`Executor` decoupled from `Tui`** — audit approval injected as
`std::function<bool(const ToolCall&)>` callback. `Chat` passes a lambda
calling `tui.request_tool_audit()`; `Runner` passes a lambda reading from
stdin (auto-reject when not a tty).

**`Tui::request_tool_audit()`** — new non-pure virtual method (default returns
`true`) so existing `MockTui` stubs do not break.

**Built-ins compiled into `ur_lib`** — not shipped as `.so` files. Same `Tool`
ABC as plugins; `Executor` cannot distinguish built-ins from plugins. External
plugin with the same name overrides a built-in (override semantics in `Loader`).

**`web_search` backend — SearXNG** — HTTP GET against
`$UR_SEARCH_BASE_URL/search?q=<query>&format=json`. User runs SearXNG
externally (Docker recommended). Tool is skipped at registration with a warning
if `UR_SEARCH_BASE_URL` is unset. No Python binding; no subprocess.

**`web_fetch` content parsing** — simple in-process HTML stripping for Phase 4.
`UR_FETCH_READER_URL` optionally forwards to a self-hosted reader service
(Trafilatura, Readability, etc.). Jina Reader rejected — not self-hostable.

**`run_agent` dropped** — `bash` tool already subsumes it. Multi-agent
delegation deferred to Phase 7 with proper design.

**`read_image`** — validates path and returns it as-is; llama.cpp handles
encoding. No base64 in `ur`.

**CMakeLists** — switched from explicit source list to `file(GLOB_RECURSE ...
CONFIGURE_DEPENDS)` per subdirectory. `libdl` linked on Linux for `dlopen`.
