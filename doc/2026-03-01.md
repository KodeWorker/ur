# Phase 2 Plan: Tool Dispatch

## Overview

Phase 2 gives the agent the ability to call tools (functions) and loop until the LLM stops
requesting tool calls. The existing `for _ in range(max_iterations)` scaffold in `loop.py`
already anticipates this ŌĆö the hard `break` is replaced with real dispatch logic.

No DB schema changes are needed: `tool_calls TEXT`, `tool_call_id TEXT`, and `name TEXT`
columns already exist in the `messages` table.

---

## Action Items

### Item 1 ŌĆö Tool Registry

**New file:** `src/ur/tools/registry.py`
**New file:** `src/ur/tools/__init__.py` (empty namespace package)

A `ToolRegistry` class holding `{name ŌåÆ (fn, schema)}`. Exposes
`as_tools_list() -> list[dict]` in OpenAI function-calling format (litellm normalises all
providers to this). A `@tool` decorator registers async functions with an explicit JSON
schema argument ŌĆö no automatic schema generation from type annotations (fragile across
providers). Plain object, not a singleton; instantiated per-run like `LLMClient`.

### Item 2 ŌĆö Accumulate Tool Calls in `CompletionStream`

**Modified:** `src/ur/llm/client.py`

`_iter()` already accumulates content deltas. Extend it to collect `delta.tool_calls`
fragments by index:

```python
tool_calls_acc: dict[int, dict] = {}
...
for tc_delta in (getattr(delta, "tool_calls", None) or []):
    idx = tc_delta.index
    if idx not in tool_calls_acc:
        tool_calls_acc[idx] = {
            "id": tc_delta.id,
            "type": "function",
            "function": {"name": tc_delta.function.name, "arguments": ""},
        }
    if tc_delta.function.arguments:
        tool_calls_acc[idx]["function"]["arguments"] += tc_delta.function.arguments
```

After the loop: `self.tool_calls = list(tool_calls_acc.values())` (empty list by default).
Also expose `self.stop_reason: str | None` and `self.has_tool_calls: bool`.

Tool-call deltas are **not** yielded as `StreamChunk` ŌĆö they are an internal concern of
the loop and never reach the CLI display layer.

### Item 4 ŌĆö Agentic Dispatch Loop

**Modified:** `src/ur/agent/loop.py`

Updated signature:

```python
async def run(
    session: AgentSession,
    client: LLMClient,
    max_iterations: int,
    system_prompt: str | None = None,
    registry: ToolRegistry | None = None,
) -> AsyncGenerator[StreamChunk, None]:
```

Loop logic (pseudocode):

```python
tools_list = registry.as_tools_list() if registry else None

for iteration in range(max_iterations):
    stream = await client.stream(messages, tools=tools_list)

    async for chunk in stream:
        yield chunk          # content tokens only; tool-call deltas stay internal

    if stream.has_tool_calls:
        session.add_assistant_tool_call_message(
            stream.tool_calls, content=stream.full_text or None
        )
        session.usage.add(stream.usage)

        for tc in stream.tool_calls:
            result = await _execute_tool(registry, tc)
            session.add_tool_result_message(tc["id"], tc["function"]["name"], result)

        messages = [{"role": "system", "content": system_prompt}, *session.messages] \
                   if system_prompt else session.messages
    else:
        session.add_assistant_message(stream.full_text)
        session.usage.add(stream.usage)
        break
```

Private `_execute_tool(registry, tool_call) -> str` coroutine: looks up function by name,
parses arguments JSON, calls the async function, returns string result. Wraps execution in
`try/except Exception` and returns an error string to the model ŌĆö tool exceptions never
propagate out of the loop.

`registry=None` preserves the exact Phase 1 behavior (immediate `else` branch, one
iteration, hard break).

### Item 5 ŌĆö `AgentSession` Updates

**Modified:** `src/ur/agent/session.py`

**(a)** Change `add_assistant_tool_call_message` to accept `list[dict]` directly (the LLM
can request multiple tool calls in one turn). Breaking change ŌĆö one existing test to update.

**(b)** Add new method:

```python
def add_tool_result_message(
    self,
    tool_call_id: str,
    name: str,
    content: str,
) -> None:
    self.messages.append({
        "role": "tool",
        "tool_call_id": tool_call_id,
        "name": name,
        "content": content,
        "created_at": _now_iso(),
    })
```

This produces a `ToolMessage` dict matching the existing TypedDict in `models.py` and
already handled correctly by `_serialize_message` in `session_store.py`.

### Item 6 ŌĆö Built-in Tools

**New file:** `src/ur/tools/builtin.py`

Starter tools using already-listed `[tools]` optional dependencies:

| Tool | Description | Deps |
|---|---|---|
| `shell` | Run a shell command, return stdout+stderr (30 s timeout, 4000 char cap) | stdlib only |
| `read_file` | Read a file path, return contents with line limit | `aiofiles` |
| `write_file` | Write content to a file path | `aiofiles` |
| `http_get` | Fetch a URL, return response body text | `httpx` |

Each tool validates its arguments explicitly and returns a clear error string on bad input
rather than raising. `create_default_registry() -> ToolRegistry` factory function
instantiates and registers all built-ins. `builtin.py` imports `registry.py`; `registry.py`
does not import `builtin.py`.

### Item 7 ŌĆö Wire Registry into CLI

**Modified:** `src/ur/main.py`

In both `_run` and `_chat`:

```python
try:
    from .tools.builtin import create_default_registry
    registry = create_default_registry()
except ImportError:
    registry = None
```

Graceful degradation: if the user has not installed the `[tools]` extra, `registry` is
`None` and the agent behaves exactly as in Phase 1. Add a `--no-tools` CLI flag on `run`
and `chat` that forces `registry = None` for debugging.
