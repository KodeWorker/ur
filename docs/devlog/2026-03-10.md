## Design note ‚Äî reasoning display deferred to Phase 3

- `HttpProvider::complete()` returns only `choices[0]["message"]["content"]`
- Thinking models (DeepSeek-R1, QwQ) put reasoning in `reasoning_content`; some backends embed it in `<think>` tags inside `content`
- Phase 3 will extend the response type to carry optional reasoning and render it in the chat TUI as a distinct block
- Reasoning is ephemeral ‚Äî not stored in the database
- Tracked in `docs/plan/phase3.md` under "Reasoning Display"
