# Code Review — Phase 2 Round 2

Automated code review of the Phase 2 tool suite (second pass, after Round 1 fixes).
Findings tracked as GitHub issues in milestone **Phase 2 — Code Review Round 2**.

---

## Major

### [1] Zombie subprocess on timeout in `shell` tool
`bug` `performance`

`proc.kill()` sends SIGKILL but never awaits `proc.wait()`. The child process becomes a
zombie that persists until the event loop exits. In long-running `ur chat` sessions with
repeated timed-out commands this silently leaks OS resources.

**Location:** `src/ur/tools/builtin.py`, `shell()` timeout handler.

**Fix:**
```python
except TimeoutError:
    proc.kill()
    await proc.wait()
    return f"Error: command timed out after {timeout}s"
```

---

### [2] DNS rebinding not blocked by `_validate_url`
`security`

`_validate_url` correctly rejects raw IP literals that are private/loopback/link-local.
However any hostname that resolves to a private address (e.g. `169.254.169.254.nip.io`)
bypasses the guard entirely — `ipaddress.ip_address(host)` raises `ValueError` for
hostnames and the function returns early without resolving the name.

Complete mitigation requires a custom `httpx` transport or dual-resolution (resolve →
check → connect). For `browser_get`, Playwright makes its own DNS resolution which is
entirely outside `_validate_url`'s control.

**Location:** `src/ur/tools/builtin.py`, `_validate_url()` lines 28–33.

**Fix:** At minimum document the known limitation with a comment. Consider a partial
hostname denylist for common dynamic-DNS bypass patterns (`.nip.io`, `.xip.io`,
`.sslip.io`).

---

### [3] Rich markup injection in `TurnWidget` and `add_error`
`bug` `security`

`TurnWidget.compose` embeds `self._user_text` directly into a `markup=True` Static
without escaping. If the user types `[red]oops[/red]` or a malformed tag, Rich will
either render unintended markup or raise `MarkupError` and crash the widget.
The same issue appears in `add_error` where `str(e)` (which may contain
attacker-controlled content from a tool result) is embedded in a `markup=True` string.

**Location:** `src/ur/tui.py` — `TurnWidget.compose` and `add_error`.

**Fix:** Escape user / exception text before interpolation:
```python
from rich.markup import escape
# in compose:
f"[bold blue]you[/bold blue] › {escape(self._user_text)}"
# in add_error:
f"[red]Error:[/red] {escape(text)}"
```

Note: `ToolConfirmWidget` already uses `markup=False` for tool name and args — that
widget is not affected.

## Minor

### [4] `0.0.0.0` / `::` (unspecified address) not blocked
`security`

`0.0.0.0` and `::` are not caught by `is_loopback`, `is_private`, or `is_link_local`.
On Linux, HTTP requests to `http://0.0.0.0/` typically reach `127.0.0.1`.

**Location:** `src/ur/tools/builtin.py`, `_validate_url()` line 32.

**Fix:** Add `addr.is_unspecified` to the condition:
```python
if addr.is_loopback or addr.is_private or addr.is_link_local or addr.is_unspecified:
```

### [5] `settings.log_level` field ignored by `_configure_logging`
`bug`

`Settings` exposes `log_level: str = Field(default="INFO")` (configurable via
`UR_LOG_LEVEL`) but `_configure_logging` hard-codes `logging.WARNING` for the root
logger and never reads the setting. The handler level is also never set (defaults to
`NOTSET`). `UR_LOG_LEVEL=DEBUG` has no effect.

**Location:** `src/ur/main.py`, `_configure_logging()`.

**Fix:**
```python
level = getattr(logging, settings.log_level.upper(), logging.WARNING)
handler.setLevel(level)
if not logging.root.level:
    logging.root.setLevel(level)
```

### [6] `assert` in `UrApp.on_mount` stripped by `-O`
`code-quality`

`assert self._run_task is not None` in `on_mount` is a programming invariant (not a
test assertion) but is silently stripped when Python runs with `-O`, producing an
unhelpful `AttributeError` on `None` instead of a clear message.

**Location:** `src/ur/tui.py`, `UrApp.on_mount`.

**Fix:**
```python
if self._run_task is None:
    raise RuntimeError("run mode requires a task string")
```

### [7] `launch_chat` eagerly creates a workspace dir every session
`performance`

Both `launch_run` and `launch_chat` create a per-session workspace at
`data_dir/workspaces/<session-id>/` on startup. For chat sessions that never invoke a
file-writing tool this creates an empty directory that accumulates indefinitely.

**Location:** `src/ur/tui.py`, `launch_chat`.

**Fix:** Create the workspace lazily on first file-tool use, or limit eager creation to
`launch_run`. At minimum document the behaviour so users know to prune `workspaces/`.

`testing`

`code-quality`

### [10] `_load_one` does not clean up `sys.modules` on `BaseException`
`bug`

The `sys.modules` cleanup after a failed `exec_module` is inside `except Exception`.
If a plugin's top-level code calls `sys.exit()` or raises `KeyboardInterrupt`
(`BaseException` but not `Exception`), the half-initialised module remains in
`sys.modules`.

**Location:** `src/ur/tools/plugins.py`, `_load_one`.

**Fix:** Broaden to `except BaseException` and re-raise non-`Exception` types after
cleanup:
```python
except BaseException as exc:
    logger.warning("Plugin %s: import failed — skipped", path, exc_info=True)
    sys.modules.pop(module_name, None)
    if not isinstance(exc, Exception):
        raise
    return
```

### [11] `test_builtin_shell_handles_error` trivially passes
`testing`

`exit 1` produces no stdout so the test asserts `isinstance(result, str)` — trivially
true for both `"(no output)"` and any error string. The test gives false confidence.

**Location:** `tests/unit/test_tools.py`, `test_builtin_shell_handles_error`.

**Fix:** Assert the exact expected sentinel: `assert result == "(no output)"`.

### [12] `_configure_logging` not covered by any test
`testing`

`_configure_logging` is never exercised by the test suite (fixtures use `tmp_settings`
directly, bypassing `_settings()`). No test verifies a `RotatingFileHandler` is
attached, the log file is created under `logs_dir`, or that calling the function twice
does not double-register handlers.

**Location:** `tests/unit/test_main.py`.

## Suggestions

### [13] `ToolRegistry` has no `dispatch` method despite documented interface
`code-quality`

The Phase 2 scope describes "ToolRegistry: register, lookup, dispatch" but the class
has `register`, `get`, `as_tools_list`, and `__contains__` — dispatch lives in
`loop._execute_tool`. Either update the docstring/architecture description or add a
thin `async def dispatch(self, tool_call: dict) -> str` that delegates to
`_execute_tool`.

---

### [14] `http_get` sends `python-httpx/<version>` User-Agent
`code-quality`

Many CDNs and bot-protection services respond to the default `httpx` User-Agent with a
403 or JavaScript challenge page. Setting a plausible browser User-Agent would improve
reliability.

**Location:** `src/ur/tools/builtin.py`, `http_get`.

## Strengths

- Lambda / `functools.partial` pattern in `create_default_registry` keeps operator
  limits entirely out of LLM-visible JSON schemas — the right separation of concerns.
- `_validate_url` covers the most important SSRF vectors and is applied consistently in
  both `http_get` and `browser_get`.
- `ToolConfirmWidget` integrates cleanly with Textual's event loop via `asyncio.Future`;
  `markup=False` on tool name and args prevents injection; `event.stop()` prevents
  bubbling.
- Plugin loader error handling covers all failure modes independently with `exc_info=True`
  logging and correct `sys.modules` cleanup for `Exception`-derived failures.
- `_stream_turn` session lifecycle correctly maps `complete()` / `fail()` /
  `interrupt()` to exit paths; `save_session` in `finally` ensures persistence.
- Tool call / result collapsible: `_active_tool_result` is `None`d after
  `add_tool_result` so a spurious second call is a no-op, not a crash.
- Test organisation is solid: loop tests cover tool dispatch round-trip,
  max-iterations termination, unknown tool, and tool exception cases.

## Code Review Round 2 — Fixes

All 14 findings addressed in PR #93 (branch `fix/phase-2-code-review-2`),
closing issues #79–#92.

### [1] → #79 — `await proc.wait()` after kill in `shell` timeout handler

`src/ur/tools/builtin.py`

Added `await proc.wait()` immediately after `proc.kill()` so the child process is
reaped before the coroutine returns. Without this, timed-out subprocesses accumulated
as zombies for the lifetime of the event loop.

```python
except TimeoutError:
    proc.kill()
    await proc.wait()
    return f"Error: command timed out after {timeout}s"
```

### [2] → #80 — Hostname denylist for DNS-rebinding bypass patterns

`src/ur/tools/builtin.py`

Added a suffix denylist (`".nip.io"`, `".xip.io"`, `".sslip.io"`) in `_validate_url`
for the common wildcard DNS services that resolve arbitrary hostnames to private IPs.
Added an inline comment documenting the remaining limitation: hostname resolution is
not performed at validation time, so a custom DNS resolver or `httpx` transport is
needed for complete protection.

```python
_DNS_REBIND_SUFFIXES = (".nip.io", ".xip.io", ".sslip.io")

def _validate_url(url: str) -> None:
    ...
    host = parsed.hostname or ""
    if any(host.endswith(s) for s in _DNS_REBIND_SUFFIXES):
        raise ValueError(f"Blocked host (DNS-rebinding risk): {host!r}")
    # NOTE: hostname→IP resolution is not performed here; a custom httpx
    # transport is required for full SSRF protection.
    ...
```

### [7] → #85 — Lazy workspace directory creation

`src/ur/tui.py`, `src/ur/tools/builtin.py`

### [10] → #88 — `except BaseException` in `_load_one` with re-raise

`src/ur/tools/plugins.py`

Broadened the import-failure handler from `except Exception` to
`except BaseException as exc`. `sys.modules` cleanup now runs even if a plugin raises
`KeyboardInterrupt` or `SystemExit`. Non-`Exception` types are re-raised after cleanup

### [14] → #92 — Browser-like `User-Agent` header for `http_get`

`src/ur/tools/builtin.py`

Added `_HTTP_HEADERS` with a realistic Chrome User-Agent. CDNs and bot-protection
services commonly return 403 or JS-challenge pages for the default `python-httpx/<v>`
User-Agent; a browser string improves reliability without other changes.

```python
_HTTP_HEADERS = {
    "User-Agent": (
        "Mozilla/5.0 (X11; Linux x86_64) "
        "AppleWebKit/537.36 (KHTML, like Gecko) "
        "Chrome/125.0.0.0 Safari/537.36"
    )
}
```

The headers dict is passed to `httpx.AsyncClient(headers=_HTTP_HEADERS)`.
